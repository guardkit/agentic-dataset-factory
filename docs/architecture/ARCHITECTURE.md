# agentic-dataset-factory — Architecture Summary

> Generated by `/system-arch` on 2026-03-16
> Refined by `/system-plan` on 2026-03-18
> Mode: setup | Pattern: Modular Monolith

## Purpose

A general-purpose, open-source pipeline for automated training dataset generation using a Player-Coach adversarial agent loop. Given a goal and source documents, it produces a validated, structured training dataset ready for fine-tuning. Domain-agnostic by design — new use cases are config changes, not code changes.

**First domain:** GCSE English tutor (AQA specification, Year 10, on-device inference on GB10).

## Users

ML engineers and domain experts who want to generate fine-tuning datasets from source documents.

## Structural Pattern

**Modular Monolith** — single-process deployment on Dell Pro Max GB10 (DGX Spark, 128GB unified memory). Simple deployment, shared ChromaDB, easy end-to-end debugging of the Player-Coach loop.

## Module Decomposition

| Module | Directory | Responsibility | Data Owned |
|--------|-----------|----------------|------------|
| Domain Config | `domains/` | GOAL.md parsing, validation, source document inventory | GOAL.md files, source documents |
| Ingestion | `ingestion/` | Docling PDF processing, chunking, ChromaDB indexing (Stage 0) | ChromaDB collections |
| Agents | `agents/` | Player and Coach factory functions via `create_deep_agent()` | Agent instances (transient) |
| Tools | `tools/` | `rag_retrieval` (ChromaDB query) + `write_output` (layer-aware JSONL routing) | Output files |
| Prompts | `prompts/` | Base prompt templates + runtime GOAL.md section injection | Prompt templates (static) |
| Entrypoint | `agent.py` | LangGraph wiring, config loading, domain selection, generation loop orchestration | agent-config.yaml, AGENTS.md, runtime state |

## Pipeline Stages

```
Stage 0: Ingest (one-time per domain)
  Docling processes sources/ -> chunks -> ChromaDB collection

Stage 1: Generate (Player-Coach loop)
  For each generation target:
    Player: retrieve chunks -> generate example -> submit to Coach
    Coach: evaluate against GOAL.md criteria -> accept or reject with JSON
    If rejected (turns < max): Player revises -> resubmit
    If accepted: write_output routes to correct output file
    If max turns: discard + log to rejected.jsonl

Stage 2: Fine-tune (out of scope — consuming project)
Stage 3: Eval (out of scope — consuming project)
```

## Technology Stack

- **Language:** Python
- **Frameworks:** LangChain DeepAgents SDK, LangGraph runtime
- **Database:** ChromaDB embedded PersistentClient (generation pipeline only)
- **Deployment:** Docker containers via docker compose on GB10
- **Observability:** LangSmith Developer tier
- **Dependencies:** Managed via `uv`
- **Ingestion:** Docling (standard + VLM modes, proven on GB10)

## Agent Model Configuration

Both Player and Coach support configurable model backends via `agent-config.yaml`:

```yaml
player:
  provider: local | anthropic | openai
  model: <model-id>
  endpoint: <url>
  temperature: 0.7

coach:
  provider: local | anthropic | openai
  model: <model-id>
  endpoint: <url>
  temperature: 0.3
```

## Output

| File | Layer | Purpose | Consumer |
|------|-------|---------|----------|
| `output/train.jsonl` | behaviour | ShareGPT format, 75% reasoning / 25% direct | Unsloth fine-tuning |
| `output/rag_index/knowledge.jsonl` | knowledge | Curriculum content for RAG | Downstream ChromaDB seeding |
| `output/rejected.jsonl` | — | Discarded examples with rejection reason + turn count | Dataset quality analysis |

## Cross-Cutting Concerns

- **Logging:** Structured JSON logging throughout (ADR-ARCH-007)
- **Secrets:** `.env` file (gitignored), `.env.example` committed
- **Validation:** GOAL.md validated at startup — fail fast on missing sections (DDR-001)
- **Restart:** Start fresh on re-run (clean output directory), with opt-in `--resume` for checkpoint recovery (ADR-ARCH-008, ADR-ARCH-010)
- **Reproducibility:** Non-deterministic with moderate temperature + Coach as quality gate (ADR-ARCH-009)
- **Progress:** Periodic structured JSON log lines at key milestones
- **LLM resilience:** Retry with exponential backoff + per-call timeout for transient failures (ADR-ARCH-010)
- **Per-target timeout:** Discard hung targets after configurable timeout, continue pipeline (ADR-ARCH-010)
- **Coach consistency:** Structured rejection schema constrains evaluation variance; Coach temperature ≤ 0.3 recommended; rejected.jsonl captures turn-by-turn feedback for consistency analysis

## Deployment Topology

```
Dell Pro Max GB10 (DGX Spark, 128GB unified memory):
  docker compose:
    vllm-coach     — Nemotron 3 Super 120B (or other local model)
    agent-loop     — Player-Coach generation pipeline
    chromadb       — Embedded in agent-loop container

MacBook Pro M2 Max:
    Development, planning, code review only
```

- `.env` — API keys and secrets
- `domains/*/sources/` — Copyrighted source PDFs
- `output/` — Generated training data (large files)
