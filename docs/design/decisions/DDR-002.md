# DDR-002: Train/Validation Split as Post-Processing

> Status: **Accepted**
> Date: 2026-03-18
> Related: ADR-ARCH-008 (start fresh), Output module

## Context

The training data format specification calls for a train/validation split (~80/20). This split could happen inline during generation (write_output randomly routes 20% to validation.jsonl) or as a post-processing step after generation completes.

## Decision

Train/validation split is a post-processing step. The generation pipeline writes all accepted examples to `train.jsonl`. A separate script (`python -m tools.split_dataset`) splits the completed file with a configurable ratio and seed.

## Rationale

- **Deterministic:** Same seed produces same split, regardless of generation order
- **Reproducible:** Re-running the split doesn't require re-running 25 hours of generation
- **Flexible:** Ratio can be adjusted without regenerating data
- **Complete dataset:** Aligns with ADR-ARCH-008 — generation produces a complete dataset, post-processing transforms it
- **Simpler write_output:** Tool doesn't need split logic or state tracking

## Alternatives Considered

- **Inline during generation:** Non-deterministic split (depends on generation order), harder to reproduce, adds complexity to write_output

## Consequences

- Requires an additional step after generation completes
- `output/train.jsonl` contains 100% of behaviour examples until split is run
- ML engineer controls split ratio and seed explicitly
