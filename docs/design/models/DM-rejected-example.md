# Data Model: Rejected Example

> Generated by `/system-design` on 2026-03-18

## Entity: RejectedExample

A discarded generation target with full rejection history, written to `output/rejected.jsonl` when a target fails all allowed turns.

  "generation_target": {

## Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `generation_target` | `object` | Yes | The target that was attempted |
| `generation_target.category` | `string` | Yes | From GOAL.md Generation Targets |
| `generation_target.topic` | `string` | Yes | Topic assigned to this target |
| `generation_target.text` | `string` | Yes | Text/subject area |
| `generation_target.type` | `string` | Yes | `reasoning` or `direct` |
| `generation_target.grade_target` | `int \| null` | Yes | Target grade level |
| `turns` | `int` | Yes | Number of turns attempted (always == max_turns) |
| `final_score` | `int` | Yes | Score from the last Coach evaluation |
| `rejections` | `array[CoachVerdict]` | Yes | Full Coach verdict for each turn |
| `last_example` | `TrainingExample` | Yes | The final attempted example (for debugging) |
| `timestamp` | `string (ISO 8601)` | Yes | When the target was discarded |

## Purpose

- **Quality analysis:** ML engineer reviews rejection patterns to identify systematic failures
- **Prompt improvement:** Common issues in `rejections[].issues` inform Player/Coach prompt tuning
- **Metrics:** Rejection rate per category/topic/text tracks dataset generation quality
- **Debugging:** `last_example` shows what was rejected, enabling manual inspection

## Invariants

- `turns == len(rejections)` — one verdict per turn
- `turns == max_turns` from agent-config.yaml (currently 3)
- All verdicts have `decision == "revise"` (if any had `accept`, the example wouldn't be rejected)
- `final_score == rejections[-1].score`
