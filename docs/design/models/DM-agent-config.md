# Data Model: Agent Config

> Generated by `/system-design` on 2026-03-18

## Entity: AgentConfig

Unified configuration file replacing the exemplar's `coach-config.yaml`. Configures both Player and Coach agents, generation parameters, chunking, and logging.

## File: `agent-config.yaml`

```yaml
# Which domain to generate for
domain: gcse-english-tutor

# Player agent model configuration
player:
  provider: local                    # local | anthropic | openai
  model: nemotron-3-super-120b      # Model identifier
  endpoint: http://localhost:8000/v1 # Required for local provider
  temperature: 0.7                   # Generation temperature

# Coach agent model configuration
coach:
  provider: local                    # local | anthropic | openai
  model: nemotron-3-super-120b      # Model identifier
  endpoint: http://localhost:8000/v1 # Required for local provider
  temperature: 0.3                   # Lower for consistent evaluation

# Generation loop parameters
generation:
  max_turns: 3                       # Player-Coach cycles before discard

# Ingestion chunking parameters
chunking:
  chunk_size: 512                    # Tokens per chunk
  overlap: 64                        # Token overlap between chunks

# Logging configuration
logging:
  level: INFO                        # DEBUG | INFO | WARNING | ERROR
  format: json                       # json (ADR-ARCH-007)
```

## Fields

### Top-Level

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `domain` | `string` | Yes | — | Domain directory name under `domains/` |

### ModelConfig (player, coach)

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `provider` | `string` | Yes | — | `local`, `anthropic`, or `openai` |
| `model` | `string` | Yes | — | Model identifier |
| `endpoint` | `string` | If local | `""` | API endpoint URL |
| `temperature` | `float` | No | Player: 0.7, Coach: 0.3 | Sampling temperature |

### GenerationConfig

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `max_turns` | `int` | No | `3` | Max Player-Coach cycles before discard (DDR-003) |

### ChunkingConfig

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `chunk_size` | `int` | No | `512` | Tokens per chunk |
| `overlap` | `int` | No | `64` | Token overlap between chunks |

### LoggingConfig

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `level` | `string` | No | `INFO` | Log level |
| `format` | `string` | No | `json` | Log format (always json per ADR-ARCH-007) |

## Validation Rules

1. `domain` must correspond to an existing directory under `domains/`
2. `provider` must be one of: `local`, `anthropic`, `openai`
3. If `provider == "local"`, `endpoint` is required and must be a valid URL
4. `temperature` must be between 0.0 and 2.0
5. `max_turns` must be >= 1
6. `chunk_size` must be > 0
7. `overlap` must be >= 0 and < `chunk_size`

## Migration from Exemplar

The exemplar uses `coach-config.yaml` with Coach-only configuration. This design extends it to:
- Add `domain` field for domain selection
- Add `player` section (mirrors Coach config structure)
- Add `generation` section for loop parameters
- Add `chunking` section for ingestion parameters
- Rename file to `agent-config.yaml` to reflect broader scope
