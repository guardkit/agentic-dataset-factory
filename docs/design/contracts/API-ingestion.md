# API Contract: Ingestion

> Module: `ingestion/`
> Generated by `/system-design` on 2026-03-18

## Responsibility

One-time pre-processing step per domain. Converts source documents into queryable chunks in ChromaDB. Runs before the generation pipeline as a separate CLI command.

## CLI Interface

```bash
python -m ingestion.ingest --domain <domain-name>
```

### Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `--domain` | Yes | Domain directory name under `domains/` |
| `--chunk-size` | No | Override chunk size (default: from agent-config.yaml, fallback 512) |
| `--overlap` | No | Override chunk overlap (default: from agent-config.yaml, fallback 64) |
| `--force` | No | Re-ingest even if collection already exists |

### Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success — collection created and populated |
| 1 | Error — domain directory not found |
| 2 | Error — GOAL.md missing or Source Documents section invalid |
| 3 | Error — Docling processing failure |
| 4 | Error — ChromaDB indexing failure |

## Processing Flow

```
1. Load GOAL.md → parse Source Documents table
2. For each source document matching file patterns:
   a. Determine Docling mode (standard | vlm)
   b. Process through Docling → extract text
   c. Chunk via chunker.py (fixed-size, 512 tokens / 64 overlap)
   d. Assign chunk metadata (source_file, page_number, chunk_index)
3. Create/replace ChromaDB collection named "{domain_name}"
4. Index all chunks with embeddings + metadata
5. Log summary: total documents, total chunks, collection name
```

## ChromaDB Collection Strategy

- **Collection name:** Domain directory name (e.g., `gcse-english-tutor`)
- **Lifecycle:** Created by ingestion, replaced on re-ingestion (delete + create)
- **Persistence:** `PersistentClient` — survives container restarts (ADR-ARCH-004)
- **Embedding:** ChromaDB default embedding function (all-MiniLM-L6-v2)
- **Lazy initialisation:** Collection may not exist at startup — generation pipeline checks existence

## Chunk Metadata Schema

```python
{
    "source_file": "mr-bruff-language.pdf",   # Original filename
    "page_number": 42,                         # Page in source document
    "chunk_index": 3,                          # Sequential index within page
    "docling_mode": "standard",                # Processing mode used
    "domain": "gcse-english-tutor"             # Domain name
}
```

## Python Contract: Chunker

```python
@dataclass
class Chunk:
    text: str
    metadata: dict  # source_file, page_number, chunk_index, docling_mode, domain

def chunk_text(
    text: str,
    chunk_size: int = 512,
    overlap: int = 64,
    source_metadata: dict = None,
) -> list[Chunk]:
    """Split text into fixed-size chunks with overlap.

    Uses RecursiveCharacterTextSplitter from LangChain.

    Args:
        text: Extracted text from Docling
        chunk_size: Target chunk size in tokens
        overlap: Overlap between consecutive chunks in tokens
        source_metadata: Metadata to attach to each chunk

    Returns:
        List of Chunk objects with text and metadata.
    """
```

## Python Contract: Ingest

```python
def ingest_domain(
    domain_name: str,
    chunk_size: int = 512,
    overlap: int = 64,
    force: bool = False,
) -> IngestResult:
    """Run the full ingestion pipeline for a domain.

    Args:
        domain_name: Name of domain directory under domains/
        chunk_size: Chunk size in tokens
        overlap: Chunk overlap in tokens
        force: Re-ingest even if collection exists

    Returns:
        IngestResult with statistics.

    Raises:
        DomainNotFoundError: Domain directory doesn't exist
        GoalValidationError: GOAL.md missing or Source Documents invalid
        IngestionError: Docling or ChromaDB failure
    """

@dataclass
class IngestResult:
    domain: str
    collection_name: str
    documents_processed: int
    chunks_created: int
    elapsed_seconds: float
```

## Error Handling

- Docling failures on individual files: log error, skip file, continue with remaining
- ChromaDB connection failure: fail with exit code 4
- Empty sources directory: fail with exit code 2
- All tools return error strings, never raise exceptions (D7) — applies to ingestion CLI errors only, not internal functions
