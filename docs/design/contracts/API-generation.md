# API Contract: Generation

> Module: `agents/` + `prompts/`
> Generated by `/system-design` on 2026-03-18

## Responsibility

The core pipeline — Player-Coach adversarial loop producing training examples. Sequential generation (ADR-ARCH-006), non-deterministic with Coach quality gate (ADR-ARCH-009).

## Player-Coach Protocol

```
For each GenerationTarget from GOAL.md:
  turn = 0
  while turn < max_turns (default 3):
    turn += 1

    Player:
      1. Receive generation target (category, type, text, grade_target)
      2. Call rag_retrieval to get relevant curriculum chunks
      3. Generate training example (ShareGPT format with metadata)
      4. Submit example to Coach for evaluation

    Coach:
      1. Receive example + GOAL.md evaluation criteria
      2. Evaluate against all criteria
      3. Return structured JSON verdict

    If Coach decision == "accept":
      Player calls write_output with accepted example
      Break → next target

    If Coach decision == "revise":
      Player receives rejection feedback
      Continue → next turn with revision context

  If turn == max_turns and still rejected:
    Write to rejected.jsonl with full rejection history
    Continue → next target
```

## Prompt Injection Design

### Player Prompt Structure

```
[Base Player Prompt]
  - Role: training data generator
  - Instructions: generate ShareGPT examples
  - Tool usage guidance (rag_retrieval, write_output)
  - Output format requirements

[Injected from GOAL.md]
  - ## Goal (what behaviour to teach)
  - ## System Prompt (to embed in every example)
  - ## Generation Guidelines (style, pedagogy, constraints)
  - ## Output Schema (JSON structure to produce)
  - ## Metadata Schema (valid field values)
  - ## Layer Routing (behaviour vs knowledge classification)
```

### Coach Prompt Structure

```
[Base Coach Prompt]
  - Role: quality evaluator
  - Instructions: evaluate against criteria, return structured JSON
  - Response format: Coach Rejection Schema (see below)
  - Scoring guidance

[Injected from GOAL.md]
  - ## Goal (what behaviour is expected)
  - ## Evaluation Criteria (rubric with named criteria)
  - ## Output Schema (what a valid example looks like)
  - ## Metadata Schema (valid metadata values to check)
  - ## Layer Routing (to validate layer classification)
```

## Coach Rejection Schema

The Coach returns this JSON structure for every evaluation:

```json
{
  "decision": "accept | revise",
  "score": 1,
  "layer_correct": true,
  "type_correct": true,
  "criteria_met": {
    "socratic_approach": true,
    "ao_accuracy": false,
    "mark_scheme_aligned": true,
    "age_appropriate": true,
    "factual_accuracy": false
  },
  "issues": [
    {
      "criterion": "ao_accuracy",
      "severity": "blocking",
      "description": "AO3 cited but response contains no contextual analysis",
      "suggestion": "Add reference to Victorian social attitudes or Dickens' purpose"
    },
    {
      "criterion": "factual_accuracy",
      "severity": "minor",
      "description": "Incorrect act reference — soliloquy is Act 1 Scene 7, not Act 2",
      "suggestion": "Correct the act/scene reference"
    }
  ],
  "quality_assessment": "Strong pedagogical approach but factual errors undermine credibility"
}
```

### Field Definitions

| Field | Type | Description |
|-------|------|-------------|
| `decision` | `"accept" \| "revise"` | Whether the example passes quality gate |
| `score` | `integer (1-5)` | Overall quality score. 1=poor, 3=adequate, 5=excellent |
| `layer_correct` | `boolean` | Whether `metadata.layer` is correctly classified |
| `type_correct` | `boolean` | Whether `metadata.type` matches `<think>` block presence |
| `criteria_met` | `dict[str, bool]` | Per-criterion pass/fail. Keys match GOAL.md Evaluation Criteria names |
| `issues` | `array` | Specific problems found (empty array if accepted) |
| `issues[].criterion` | `string` | Which evaluation criterion this issue relates to |
| `issues[].severity` | `"blocking" \| "minor"` | Whether this issue alone would cause rejection |
| `issues[].description` | `string` | What is wrong |
| `issues[].suggestion` | `string` | How the Player should fix it |
| `quality_assessment` | `string` | Free-text overall assessment |

### Acceptance Rule

An example is accepted when:
- `decision == "accept"`
- `score >= 3`
- `layer_correct == true`
- `type_correct == true`
- No `blocking` severity issues

## Generation Target Assignment

The entrypoint reads `## Generation Targets` from GOAL.md and produces a flat list of generation targets:

```python
@dataclass
class GenerationTarget:
    category: str           # e.g., "Literary analysis (single-turn)"
    type: str               # "reasoning" or "direct"
    text: str               # e.g., "macbeth" (distributed per Text Coverage)
    grade_target: int | None  # e.g., 7 (distributed across grade range)
    topic: str              # e.g., "character_analysis"
```

The entrypoint iterates over targets sequentially (ADR-ARCH-006).
