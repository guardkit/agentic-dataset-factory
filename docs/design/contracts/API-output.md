# API Contract: Output

> Module: `output/`
> Generated by `/system-design` on 2026-03-18

## Responsibility

File-based output consumed by downstream projects. Cleaned on each run (ADR-ARCH-008). No runtime API — downstream projects read JSONL files directly.

## Output Directory Structure

```
output/                          ← cleaned at startup (ADR-ARCH-008)
├── train.jsonl                  ← behaviour layer examples
├── rejected.jsonl               ← discarded examples with rejection history
└── rag_index/
    └── knowledge.jsonl          ← knowledge layer examples
```

## File 1: `train.jsonl` — Behaviour Layer

One JSON object per line. ShareGPT format for Unsloth SFTTrainer.

```json
{
  "messages": [
    {"role": "system", "content": "<system prompt from GOAL.md>"},
    {"role": "user", "content": "<student question>"},
    {"role": "assistant", "content": "<think>\n<reasoning>\n</think>\n\n<response>"}
  ],
  "metadata": {
    "layer": "behaviour",
    "type": "reasoning",
    "ao": ["AO1", "AO2"],
    "text": "macbeth",
    "topic": "character_analysis",
    "grade_target": 7,
    "source": "synthetic",
    "turns": 1
  }
}
```

### Constraints

- 75% of examples must have `type: "reasoning"` (with `<think>` blocks)
- 25% must have `type: "direct"` (no `<think>` blocks)
- This ratio is enforced by Generation Targets in GOAL.md, not by write_output
- Multi-turn examples have multiple user/assistant pairs and `turns > 1`

## File 2: `rag_index/knowledge.jsonl` — Knowledge Layer

Same JSON structure as train.jsonl. Contains examples where `metadata.layer == "knowledge"`.

These examples seed ChromaDB at inference time (downstream concern — consumed by study-tutor-factory).

### Purpose

- ML engineer reviews rejected examples to identify systematic failure patterns
- Informs prompt improvement for Player and Coach
- Tracks rejection rate per category/topic/text for dataset quality analysis

## Post-Processing: Train/Validation Split

The train/validation split is a **post-processing step**, not part of the generation pipeline.

```bash
python -m tools.split_dataset \
  --input output/train.jsonl \
  --train-ratio 0.8 \
  --seed 42
```

Produces:
- `output/train.jsonl` — 80% of examples (overwritten)
- `output/validation.jsonl` — 20% held out

This is run after generation completes, ensuring a deterministic, reproducible split.

## Consumers

| File | Consumer | Purpose |
|------|----------|---------|
| `train.jsonl` | Unsloth (downstream) | QLoRA fine-tuning of Nemotron 3 Nano |
| `knowledge.jsonl` | study-tutor-factory (downstream) | ChromaDB seeding for RAG at inference |
| `rejected.jsonl` | ML engineer (human) | Quality analysis, prompt improvement |
| `validation.jsonl` | Eval pipeline (downstream) | Post-fine-tuning evaluation |
