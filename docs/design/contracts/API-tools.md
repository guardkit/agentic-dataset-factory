# API Contract: Tools

> Module: `tools/`
> Generated by `/system-design` on 2026-03-18

## Responsibility

Two LangChain `@tool` decorated functions used exclusively by the Player agent. Coach has no tools (D5). All tools return error strings, never raise exceptions (D7).

## Tool 1: `rag_retrieval`

### Contract

```python
@tool
def rag_retrieval(query: str, n_results: int = 5) -> str:
    """Retrieve curriculum chunks from ChromaDB relevant to the generation target.

    Args:
        query: Natural language query describing what content to retrieve.
               e.g., "Macbeth Act 1 Scene 7 ambition soliloquy"
        n_results: Number of chunks to return (default 5, max 20)

    Returns:
        Formatted string of retrieved chunks with source metadata.
        Format:
            --- Chunk 1 (source: mr-bruff-language.pdf, p.42) ---
            <chunk text>

            --- Chunk 2 (source: aqa-mark-schemes/paper1.pdf, p.12) ---
            <chunk text>

        On error: Returns error string describing the failure.
        e.g., "Error: ChromaDB collection 'gcse-english-tutor' not found"
    """
```

### Internal Behaviour

1. Connect to ChromaDB `PersistentClient` (lazy initialisation)
2. Get collection by domain name (injected at tool creation time)
3. Query with `query` text, return top `n_results` by similarity
4. Format chunks with source metadata headers
5. On any error: return descriptive error string

### Configuration

The collection name is injected when the tool is created (not passed by the agent):

```python
def create_rag_retrieval_tool(collection_name: str) -> Callable:
    """Factory that creates a rag_retrieval tool bound to a specific collection."""
```

## Tool 2: `write_output`

### Contract

```python
@tool
def write_output(example_json: str) -> str:
    """Validate and write an accepted training example to the correct output file.

    Routes by metadata.layer field:
      - "behaviour" → output/train.jsonl
      - "knowledge" → output/rag_index/knowledge.jsonl

    Args:
        example_json: Complete training example as JSON string, conforming to
                      GOAL.md Output Schema. Must include both 'messages' and
                      'metadata' top-level keys.

    Returns:
        Success: "Written to output/train.jsonl (example #42)"
        Error: Descriptive error string.

    Possible errors:
        "Error: Invalid JSON"
        "Error: Missing required field 'messages'"
        "Error: Missing required field 'metadata'"
        "Error: Invalid metadata.layer value 'unknown' (expected: behaviour, knowledge)"
        "Error: metadata.type is 'reasoning' but assistant content has no <think> block"
        "Error: metadata.type is 'direct' but assistant content contains <think> block"
        "Error: messages[0].role must be 'system'"
        "Error: metadata.text value 'hamlet' not in valid values"
    """
```

### Validation Checks (in order)

1. Parse JSON string → reject if invalid JSON
2. Check `messages` key exists and is a non-empty array
3. Check `messages[0].role == "system"`
4. Check `metadata` key exists and is an object
5. Check `metadata.layer` is `"behaviour"` or `"knowledge"`
6. Check `metadata.type` is `"reasoning"` or `"direct"`
7. If `type == "reasoning"`: verify last assistant message contains `<think>` block
8. If `type == "direct"`: verify last assistant message does NOT contain `<think>` block
9. Validate all metadata fields against GOAL.md Metadata Schema valid values
10. Append validated JSON line to the correct output file

### Configuration

The output directory and metadata schema are injected at tool creation time:

```python
def create_write_output_tool(
    output_dir: Path,
    metadata_schema: list[MetadataField],
) -> Callable:
    """Factory that creates a write_output tool bound to output config."""
```

### Layer Routing

| `metadata.layer` | Output file | Purpose |
|-------------------|-------------|---------|
| `behaviour` | `{output_dir}/train.jsonl` | Fine-tuning dataset (ShareGPT) |
| `knowledge` | `{output_dir}/rag_index/knowledge.jsonl` | RAG index seed |

### Write Mode

- **Append mode** — each accepted example is appended as a single JSON line
- **Atomic writes** — each line is flushed immediately (no buffering)
- **No concurrent writes** — sequential generation (ADR-ARCH-006) guarantees single writer
