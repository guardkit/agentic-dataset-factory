## Part 2: Deploy Model to MacBook Pro

### 2.1 Copy GGUF from DGX Spark to MacBook via Tailscale + rsync

The model files on the DGX Spark (inside Docker or on the host) are at:
```
/workspace/output/gcse-tutor-gemma4-26b-moe/gguf_gguf/
```

The key files to transfer:
- `gemma-4-26b-a4b-it.Q4_K_M.gguf` - the quantised model (~15 GB estimated)
- `gemma-4-26b-a4b-it.BF16-mmproj.gguf` - multimodal projector (if needed)
- `Modelfile` - Ollama Modelfile generated by Unsloth

#### Step-by-step: rsync over Tailscale

1. **Ensure both machines are on Tailscale**
   ```bash
   # On MacBook
   tailscale status
   # Confirm the DGX Spark hostname (e.g., dgx-spark or its Tailscale IP)
   ```

2. **Copy GGUF files from DGX Spark to MacBook**

   If the files are inside the Docker container, first copy them to the host:
   ```bash
   # On DGX Spark host
   docker cp <container_id>:/workspace/output/gcse-tutor-gemma4-26b-moe/gguf_gguf/ \
     ~/fine-tuning/output/gcse-tutor-gemma4-26b-moe/gguf_gguf/
   ```

   Then rsync to MacBook:
   ```bash
   # On MacBook (pulling from DGX Spark)
   mkdir -p ~/Models/gcse-tutor-gemma4-26b-moe

   rsync -avP --partial \
     <dgx-spark-tailscale-hostname>:~/fine-tuning/output/gcse-tutor-gemma4-26b-moe/gguf_gguf/gemma-4-26b-a4b-it.Q4_K_M.gguf \
     ~/Models/gcse-tutor-gemma4-26b-moe/

   rsync -avP --partial \
     <dgx-spark-tailscale-hostname>:~/fine-tuning/output/gcse-tutor-gemma4-26b-moe/gguf_gguf/Modelfile \
     ~/Models/gcse-tutor-gemma4-26b-moe/
   ```

   **Flags explained**: `-a` archive, `-v` verbose, `-P` progress + partial (resume interrupted transfers). The `--partial` flag is important for large files over Tailscale as it allows resuming.

   Alternatively, push from DGX Spark:
   ```bash
   # On DGX Spark host
   rsync -avP --partial \
     ~/fine-tuning/output/gcse-tutor-gemma4-26b-moe/gguf_gguf/gemma-4-26b-a4b-it.Q4_K_M.gguf \
     <macbook-tailscale-hostname>:~/Models/gcse-tutor-gemma4-26b-moe/
   ```

3. **Verify the transfer**
   ```bash
   # On MacBook
   ls -lh ~/Models/gcse-tutor-gemma4-26b-moe/
   # Check file size matches source
   md5sum ~/Models/gcse-tutor-gemma4-26b-moe/gemma-4-26b-a4b-it.Q4_K_M.gguf
   ```

### 2.2 Register Model in Ollama

1. **Edit the Modelfile** (if needed)

   The Unsloth-generated Modelfile at `~/Models/gcse-tutor-gemma4-26b-moe/Modelfile` should already point to the GGUF. Check and update the `FROM` path:
   ```bash
   cat ~/Models/gcse-tutor-gemma4-26b-moe/Modelfile
   ```

   If the `FROM` path is absolute and points to the DGX path, update it:
   ```bash
   # Edit Modelfile to point to local path
   sed -i '' 's|FROM .*|FROM ./gemma-4-26b-a4b-it.Q4_K_M.gguf|' \
     ~/Models/gcse-tutor-gemma4-26b-moe/Modelfile
   ```

2. **Create the Ollama model**
   ```bash
   cd ~/Models/gcse-tutor-gemma4-26b-moe
   ollama create gcse-tutor-gemma4-moe -f Modelfile
   ```

3. **Quick smoke test**
   ```bash
   ollama run gcse-tutor-gemma4-moe \
     "I'm studying Macbeth for my GCSE. Can you help me understand the significance of the dagger soliloquy?"
   ```

4. **Verify model is listed**
   ```bash
   ollama list
   ```

### 2.3 Set Up ChromaRAG

ChromaRAG provides retrieval-augmented generation using ChromaDB as the vector store, allowing the tutor model to reference GCSE specification documents, mark schemes, and exemplar material.

1. **Install ChromaRAG** (if not already installed)
   ```bash
   # Clone or install ChromaRAG
   pip install chromarag
   # Or clone the repo if it's a local project
   ```

2. **Ingest GCSE reference documents into ChromaDB**

   Prepare your reference documents (AQA specs, mark schemes, exemplar essays, poetry anthology notes) and ingest them:
   ```bash
   # Example: ingest documents into a ChromaDB collection
   chromarag ingest \
     --collection gcse-english \
     --input-dir ~/Documents/GCSE-English-Resources/ \
     --chunk-size 512 \
     --chunk-overlap 50
   ```

3. **Configure ChromaRAG to use the Ollama model**

   Set up the ChromaRAG config to point to your local Ollama instance and the tutor model:
   ```yaml
   # chromarag-config.yaml (example)
   llm:
     provider: ollama
     model: gcse-tutor-gemma4-moe
     base_url: http://localhost:11434
   
   vectorstore:
     provider: chromadb
     collection: gcse-english
     persist_directory: ~/.chromarag/gcse-english
   
   retrieval:
     top_k: 5
     score_threshold: 0.7
   ```

4. **Test RAG retrieval**
   ```bash
   chromarag query \
     --collection gcse-english \
     "What are the assessment objectives for AQA English Literature Paper 1?"
   ```

### 2.4 Wire Up Open WebUI

Open WebUI provides a ChatGPT-style web interface for interacting with the tutor model, with RAG support.

1. **Install/Start Open WebUI** (if not already running)
   ```bash
   # Docker method (recommended)
   docker run -d \
     --name open-webui \
     -p 3000:8080 \
     -v open-webui:/app/backend/data \
     -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
     --add-host=host.docker.internal:host-gateway \
     --restart always \
     ghcr.io/open-webui/open-webui:main
   ```

   Or if already installed via pip:
   ```bash
   open-webui serve --port 3000
   ```

2. **Access Open WebUI**
   - Navigate to `http://localhost:3000`
   - Create an account on first launch
   - The `gcse-tutor-gemma4-moe` model should appear in the model selector (auto-detected from Ollama)

3. **Configure RAG in Open WebUI**
   - Go to **Settings > Documents** (or Admin Panel > Settings > Documents)
   - Set the embedding model (e.g., `nomic-embed-text` via Ollama)
   - Upload GCSE reference documents directly through the UI, or connect to your existing ChromaDB:
     - If using Open WebUI's built-in RAG: upload PDFs/docs via the **Documents** section
     - If using external ChromaDB: configure the connection in Settings > Documents > Vector DB settings

4. **Create a GCSE Tutor workspace/model preset**
   - In Open WebUI, go to **Workspace > Models**
   - Create a new model preset called "GCSE English Tutor"
   - Set the base model to `gcse-tutor-gemma4-moe`
   - Add a system prompt (the model already has the tutor persona baked in, but you can reinforce it):
     ```
     You are an expert GCSE English tutor supporting students studying the AQA specification.
     Use Socratic questioning to guide students to discover answers.
     Reference the uploaded documents for accurate specification details.
     ```
   - Enable RAG/document context for this preset
   - Attach relevant document collections

5. **Test the full pipeline**
   - Select "GCSE English Tutor" from the model dropdown
   - Attach a relevant document (e.g., AQA mark scheme)
   - Ask: "How should I structure a Grade 9 response to an extract question on Macbeth?"
   - Verify the response uses both the fine-tuned knowledge and retrieved document context
